# Sprint 2: 글 작성 기능

## 목표

글 CRUD 및 임시 저장 기능 구현

---

## 작업 목록

### Backend (API Server)

- [ ] 글 모델 및 레포지토리 구현
- [ ] GET /api/writings 구현 (목록 조회, 페이지네이션)
- [ ] GET /api/writings/:id 구현 (상세 조회)
- [ ] POST /api/writings 구현 (글 생성/임시 저장)
- [ ] PUT /api/writings/:id 구현 (글 수정)
- [ ] DELETE /api/writings/:id 구현 (글 삭제)
- [ ] 글 길이 검증 (최대 2,000자)
- [ ] 글 종류 검증 (essay, cover_letter)
- [ ] 본인 글만 접근 가능하도록 권한 체크

### Frontend

- [ ] 글 목록 페이지 (대시보드)
- [ ] 글 작성/수정 페이지
- [ ] 텍스트 에디터 컴포넌트 (글자 수 표시)
- [ ] 글 종류 선택 UI
- [ ] 임시 저장 기능 (자동 저장 또는 수동)
- [ ] 글 삭제 확인 모달

### 테스트

- [ ] 글 CRUD API 단위 테스트
- [ ] 권한 체크 테스트

---

## 완료 조건

- [ ] 글 생성, 조회, 수정, 삭제 가능
- [ ] 임시 저장 기능 동작
- [ ] 2,000자 초과 시 에러 반환
- [ ] 다른 사용자의 글 접근 시 403 반환
- [ ] 글 CRUD 테스트 통과
