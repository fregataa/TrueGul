# TrueGul v0 스프린트 계획

## 개요

- **목표**: MVP 출시
- **스프린트 단위**: 2주
- **총 예상 스프린트**: 6 스프린트 (Sprint 0 ~ 5)

---

## 마일스톤 요약

| 스프린트 | 목표 | 주요 산출물 |
|----------|------|-------------|
| [Sprint 0](./sprint-0.md) | 프로젝트 셋업 | 개발 환경, DB 스키마, CI 기초 |
| [Sprint 1](./sprint-1.md) | 인증 시스템 | 회원가입/로그인/로그아웃 |
| [Sprint 2](./sprint-2.md) | 글 작성 기능 | 글 CRUD, 임시 저장 |
| [Sprint 3](./sprint-3.md) | AI 분석 | ML Server, AI 분석 파이프라인 |
| [Sprint 4](./sprint-4.md) | Frontend 통합 | 분석 결과 UI, E2E 통합 |
| [Sprint 5](./sprint-5.md) | 배포 | AWS 배포, 프로덕션 준비 |

---

## 리스크 및 대응

| 리스크 | 영향 | 대응 방안 |
|--------|------|----------|
| AI 모델 추론 속도 | 10초 목표 초과 가능 | CPU 최적화 모델 선정, 필요시 GPU 검토 |
| OpenAI API 비용 | 예산 초과 | 테스트 시 목 사용, 프롬프트 최적화 |
| ECS 배포 복잡도 | 일정 지연 | Sprint 0에서 충분한 학습 시간 확보 |
| 비동기 처리 안정성 | 분석 누락 | 재시도 로직, 상태 모니터링 |

---

## 참고사항

- 각 스프린트 종료 시 회고 진행
- 스프린트 내 작업 우선순위는 Backend → ML → Frontend 순서 권장
- 테스트 작성은 각 기능 구현과 함께 진행
